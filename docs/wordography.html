Overview

Wordography is a daily geography guessing game — similar to Wordle but for country names. Players have 5 attempts to guess the target country based on geographic proximity hints.

🎯 One Puzzle Per Day

A new country target is selected every 24 hours using a deterministic daily seed.

📍 Distance Hints

After each guess, you see directional arrows and distance in km to help narrow down the answer.

🗺️ Map Reveal

The game's modal shows the target country's approximate location on a world map.

♿ Color-Blind Mode

Toggle color-blind friendly palette where green becomes blue and yellow becomes orange.

📊 Statistics

Tracks win rate, current/best streaks, guess distribution, and average distance.

📋 Share Results

Copy emoji grid results to clipboard for sharing on social media.

How to Play

Your goal is to guess the hidden country in 5 tries or fewer.

1. Enter Your Guess

Type any country name using the on-screen keyboard or your physical keyboard. Press Enter to submit.

Gameplay Mechanics

Daily Target Selection

The target country is chosen deterministically from today's date using a hash-based seed:

seed = 'wordography-' + TODAY  // e.g. 'wordography-2026-04-15'
hash = 5381  // FNV-1a hash
index = abs(hash) % countries.length  // filtered to countries with 4+ letter names

This means every player worldwide gets the same country on the same day, and the same country will always produce the same result for a given date.

Letter Evaluation (Wordle Algorithm)

The game uses a standard Wordle-style evaluation:

  1. First pass: Any letter that exactly matches the target position becomes green and is marked as used.
  2. Second pass: Remaining letters are checked against the target name. Any letter that exists in an unused position becomes yellow. Letters not in the target become grey.

Distance Calculation

After each guess, the game calculates the great-circle distance using the Haversine formula:

hav(lat1, lon1, lat2, lon2)
  → Returns distance in kilometres
  → Also returns directional index (0-7) for arrow direction

Country Name Matching

Guesses are matched against normalizedName fields (uppercase, spaces removed). The game uses the full official country name — e.g. "United States" not "USA". Multi-word country names are fully playable (e.g. "South Korea", "New Zealand").

Keyboard State (kbState)

The keyboard state tracks letter status across multiple guesses, upgraded as you play:

  • Green — confirmed letter in correct position (never downgraded)
  • Yellow — letter exists in name but not confirmed (upgraded to green if later confirmed)
  • Grey — letter not in target name

Game State Object

state = {
  guesses: [{
    name: 'KYRGYZSTAN',
    eval: ['green', 'grey', 'grey', ...],  // Letter evaluations
    km: 213,                               // Distance from target
    di: 3,                                  // Direction index (0-7)
    flag: '🇰🇬'
  }, ...],
  current: 'KYR',     // Currently typed letters
  gameOver: false,
  won: false,
  _targetIdx: 42     // Internal: index into COUNTRIES array
}

Stats Object

stats = {
  played: 0,                   // Total games played
  won: 0,                      // Games won
  currentStreak: 0,            // Consecutive wins

}

Known Issues

Tile Colour Rendering

On some devices and browsers, tile backgrounds may appear amber/orange instead of the expected green (#6AAA64) or yellow (#FFB300). This is a browser rendering/display calibration issue rather than a code bug — the game's computed colour values are correct as confirmed by Playwright automated tests.

Current mitigation: direct inline backgroundColor style applied via JavaScript, bypassing CSS class cascade entirely.

Country Data

Country names, coordinates, and facts are embedded directly in index.html. To add or modify a country, search for the relevant entry in the COUNTRIES array literal. Flags use official country code emoji (e.g. 🇰🇬 for Kyrgyzstan).

⚠️ Critical: Project Type — Pages vs Worker

The "Connect to Git" Trap

When recreating the Cloudflare Pages project, ensure you use "Connect to Git" from the Pages section, NOT from the Workers section. If you accidentally create it as a Worker, the URL will be wordography.darren-b14.workers.dev instead of wordography.pages.dev, and the project type will be wrong.

Correct path: Cloudflare dashboard → Workers & Pages → Create application → Pages → Connect to Git

You cannot switch from Direct Upload to Git integration (or vice versa) after creation — you must delete and recreate the project.

⚠️ mobozed.com Zone Requirement

wordography.mobozed.com can only be added as a custom domain if mobozed.com is a zone in the Cloudflare account. Currently mobozed.com is registered at reg123.co.uk with just a CNAME pointing to Cloudflare Pages — it is NOT a Cloudflare zone. This means:

  • wordography.pages.dev — always works (Cloudflare-managed)
  • wordography.mobozed.com — works via CNAME at reg123.co.uk; the "Add custom domain" flow in Cloudflare dashboard may fail with "zone not found"

To enable full custom domain management in Cloudflare, add mobozed.com as a site in the Cloudflare dashboard (Add a site → enter mobozed.com → add the TXT record Cloudflare provides at reg123.co.uk to verify ownership).
