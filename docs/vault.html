Overview

Vaultwarden is a self-hosted, Bitwarden-compatible password manager running on Horizon at https://192.168.0.2:8443. It provides secure, local credential storage for all MoBoZed services — eliminating the need to store passwords in docs or memory files.

Architecture

Vaultwarden runs as a Docker container on Horizon, proxied through Caddy for HTTPS:

Internet/HTTPS → Caddy (:8443, self-signed cert) → Vaultwarden (:80)
         ↓
    /mnt/Content/docker-data/vaultwarden/data/  (on NAS)

Docker Compose

# Location: /home/darren/vaultwarden/docker-compose.yml
services:
  vaultwarden:
    image: vaultwarden/server:latest
    container_name: vaultwarden
    restart: unless-stopped
    ports:
      - "8282:80"
    volumes:
      - /home/darren/vaultwarden/data:/data
    environment:
      - ADMIN_TOKEN=darren-vault-2026-04-19
      - SIGNUPS_ALLOWED=false
      - INVITATIONS_ALLOWED=false
      - WEB_VAULT_ENABLED=true

# List all credentials

First-Time Setup

If this is a fresh install and signups are disabled:

# Temporarily enable signups (run on Horizon):
ssh darren@192.168.0.2
cd /home/darren/vaultwarden
sed -i 's/SIGNUPS_ALLOWED=false/SIGNUPS_ALLOWED=true/' docker-compose.yml
docker compose up -d

# Create account at https://192.168.0.2:8443
# Then disable signups:
sed -i 's/SIGNUPS_ALLOWED=true/SIGNUPS_ALLOWED=false/' docker-compose.yml
docker compose up -d
